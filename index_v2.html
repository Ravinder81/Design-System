Overview

Design Principles

Four principles guide every decision — with APEX-native implementation at the core.

🎯

Clarity First

Every component communicates its purpose instantly. Consistent patterns reduce cognitive load in enterprise workflows.

⚖️

APEX Consistency

Always use native APEX classes. Override with tokens only — never recreate what Universal Theme already provides.

♿

Accessibility

WCAG 2.1 AA. All APEX native components carry correct ARIA roles — don't break them with structural rewrites.

📦

Plug & Play

Drop the token block into app.css and your APEX app inherits the entire design system. Zero structural changes needed.

Component 03

t-Badge

APEX Universal Theme badge classes. Used directly in IR/IG HTML Expression columns. No custom wrapper needed — just class="t-Badge t-Badge--success".

Status Badges
● Active ● Completed ● In Progress ● Pending Review ● Failed ● Rejected ● Processing ● Draft ● Published
In APEX IR — HTML Expression
-- SQL case for dynamic badge class
CASE STATUS
  WHEN 'ACTIVE'    THEN 't-Badge--success'
  WHEN 'PENDING'   THEN 't-Badge--warning'
  WHEN 'FAILED'    THEN 't-Badge--danger'
  WHEN 'PROCESSING' THEN 't-Badge--info'
  ELSE 't-Badge--default'
END AS STATUS_CLASS

-- HTML Expression column value:
<span class="t-Badge &STATUS_CLASS.">&STATUS.</span>

3-Step APEX Quick Start

Apply the entire design system to any APEX app in three steps — no custom markup required.

1
Add token block to app.css
Shared Components → Static Application Files → app.css. Paste the entire :root { } block from this file. All APEX components now inherit the token values.
2
Set primary colour in Theme Roller
Shared Components → Themes → Universal Theme → Theme Roller. Set --ut-palette-primary to #007FA3. This syncs APEX's built-in primary with our token system.
3
Use native APEX classes — no new markup
Use t-Button--hot for primary buttons, t-Badge--success in IR HTML Expressions, t-Alert--info for messages. The token overrides apply automatically.
